Target selectors.


When you use a command you have fill in co√∂rdinations or a target selector (or both). This page wil tell you how you use them.


@p.

The @p selector stands for nearest player. This is calculated from the place an command is executed, this can be the place of an command block or an entity executing the command.
It is importand to know that @p only looks for players.

@r.

The @r selector stands for random player. This is not relevant of position, it just selects a random player in the world/server.
@r also only looks for players.

@a.

The @a selector stands for all players. It does not care about position or anything, it just grabs all players in a world/server.
This selector also only looks for players.

@e.

The @e selector stands for all entity's. This is especialy usefull in combination with arguments.
@e does not only look for players.

@s.

The @s selector stands for the entity executing the command. This can be very usefull with the execute command.
This selector does not only look for players.

Arguments.

Arguments are a way to narrow down your targeted group, you implement them by putting [argument=value] after your initial selector. When you use an argument it wil only select entity's or players that meet the arguments you specified.
There are a lot of arguments but some of the most used are: type, distance and limit.


If you want to learn more about target selectors see the wiki page about target selectors.
